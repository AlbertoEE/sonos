# Guía de diseño de sets de habilidades

Un set es el kit completo de un jugador: **2 fundamentos** (pasivas, coste 0) + **4 rudimentos** (activas, cuestan Resonancia). Tienen que funcionar juntos como las habilidades de un campeón de MOBA — que se note que alguien las diseñó pensando en las otras cinco.

---

## Lo más importante

**Es mejor que las habilidades sean fuertes.** En un TTRPG siempre es más fácil subir la dificultad de los encuentros que hacer que los jugadores se sientan inútiles. Una habilidad que emociona cuando se usa es mejor diseño que una habilidad "equilibrada" que nadie quiere tirar.

**La simplicidad es una feature.** Si necesitas más de dos frases para explicar cómo funciona una habilidad en mesa, es demasiado compleja. Las mejores habilidades se entienden en la primera lectura y sorprenden en la primera tirada.

---

## Los roles son atributos, no solo arquetipos

Cada rol en Sonos es uno de los **6 atributos del Cantor**. Esto es importante porque cada atributo tiene dos caras: lo que hace el personaje **en combate** y quién es **fuera de él**. Un set bien diseñado debe evocar las dos.

| Atributo | En combate | Fuera de combate |
|---|---|---|
| **Ritmo** | Potencia física, impacto directo, resistencia | Fuerza bruta, esfuerzo físico, determinación |
| **Armonía** | Control, protección, equilibrio | Empatía, liderazgo sereno, cuidado de los demás |
| **Melodía** | Precisión, creatividad, expresividad | Sensibilidad artística, percepción fina, intuición |
| **Disonancia** | Caos, ruptura, efectos oscuros | Temeridad, pulsiones destructivas, vivir al límite |
| **Síncopa** | Trucos, engaño, manipulación | Astucia, lectura de intenciones, nunca de frente |
| **Tempo** | Rapidez de reacción, iniciativa | Reflejos mentales, rapidez de decisión, anticipación |

**Lo que esto significa para el diseño:**

Un set de Disonancia no es solo "mecánicas de alto riesgo" — es un personaje que vive del caos, que apuesta cuando los demás se cubren, que acepta consecuencias negativas porque la alternativa es aburrida. Las habilidades deben transmitir esa personalidad, no solo el efecto mecánico.

Un set de Armonía no es solo "curación y escudos" — es alguien que antepone a los demás, cuyas habilidades más potentes requieren o benefician a aliados. El set debería sentirse incompleto en solitario.

Un set de Síncopa no ataca de frente. Sus habilidades deben tener condiciones de activación (reacciones, objetivos ya dañados, movimiento) que reflejan que este personaje espera el momento exacto.

**Cuando el set abarca dos atributos**, la tensión entre ambos puede ser el tema: un Ritmo + Disonancia es alguien que golpea fuerte pero a veces se pasa, que mezcla la contundencia con la imprudencia.

## Cómo construir un set

### 1. Empieza por la mecánica central
Antes de escribir ninguna habilidad, define en una frase qué hilo las une. Ejemplos:
- *"Acumular cargas para liberar un golpe devastador"*
- *"Marcar a un objetivo y castigarlo con cada acción posterior"*
- *"Crear zonas en el tablero que potencian las habilidades dentro de ellas"*

Si no puedes describir la conexión en una frase, el set no tiene identidad todavía.

### 2. Los fundamentos hacen el trabajo sucio
Las pasivas son las que dan personalidad al set sin que el jugador tenga que pensar en activarlas. Un buen fundamento:
- Es relevante en casi cualquier turno, no solo en situaciones específicas
- Potencia los rudimentos del mismo set de alguna manera
- Tiene un efecto claro que el jugador puede anticipar

### 3. Los rudimentos tienen que sentirse diferentes entre sí
Cada uno de los cuatro debe hacer algo que los otros tres no hacen. Una manera fácil de comprobarlo: si puedes resumir dos rudimentos con la misma frase, uno de los dos sobra.

### 4. El coste debe sentirse justo
El impacto de una habilidad tiene que estar alineado con su coste de Resonancia:

| Coste | Expectativa |
|---|---|
| 0 | Útil pero limitado — bueno para encadenar |
| 1–2 | El rango normal. Efecto claro y satisfactorio |
| 3–4 | Tiene que cambiar el estado del combate cuando se usa |
| M / X | Potente y flexible — justifica el coste variable |

Si una habilidad de coste 3 se siente igual que una de coste 1, hay un problema de diseño.

---

## Qué hacer ✓

- **Un combo obvio.** Tiene que existir una secuencia natural de 2-3 habilidades que el jugador quiera ejecutar. Si no hay un combo, el set se siente como habilidades sueltas.
- **Al menos un rudimento de coste 0 o 1.** Para que el set sea jugable cuando la Resonancia escasea.
- **Nombres con sabor.** Los nombres son parte del juego. "Eco Resonante" es mejor que "Ataque de Área 2". El universo Sonos tiene vocabulario musical — úsalo.

## Qué no hacer ✗

- **No pongas condiciones dentro de condiciones.** "Si el objetivo tiene el estado X y tú tienes más de 2 de Resonancia y es tu turno..." nadie va a acordarse de eso en mesa.
- **No hagas que dos habilidades hagan lo mismo.** Si tienes dos rudimentos que básicamente son "inflige daño a un objetivo cercano", uno de los dos no está haciendo su trabajo.
- **No equilibres desde el miedo.** Una habilidad no tiene que ser débil para ser "justa". Confía en que los jugadores se diviertan más siendo poderosos y ajusta los encuentros si es necesario.
- **No mezcles más de dos roles si no tienes claro el hilo.** Dos roles bien integrados funcionan. Tres roles en el mismo set suelen dar como resultado una identidad borrosa.

---

## Lista de comprobación antes de publicar un set

- [ ] Puedo describir la mecánica central en una frase
- [ ] Los dos fundamentos son relevantes en turnos normales, no solo en edge cases
- [ ] Cada rudimento hace algo que los otros tres no hacen
- [ ] Existe al menos un combo de 2-3 habilidades que el jugador querrá ejecutar
- [ ] Al menos un rudimento cuesta 0 o 1
- [ ] Los costes altos (3-4) justifican su precio en impacto
- [ ] Las descripciones se entienden en la primera lectura
